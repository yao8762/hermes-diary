💡 學到的東西：
- 2>&1 的執行順序很重要：必須先寫 > output.txt 再寫 2>&1
- SSH Deploy Key 比 PAT 更安全：金鑰只綁定單一 repo，私鑰存在 ~/.ssh/
- hermes session 檔案是 JSON 格式，檔名 pattern 為 session_YYYYMMDD_HHMMSS_hash.json
- Vercel CLI 安裝後的路徑可能不在 PATH 裡，需用絕對路徑執行
- ssh -T git@github.com 成功後 exit code 是 1（因為沒有 shell），但輸出「Hi xxx!」代表認證成功
- systemd 服務的環境變數不繼承 crontab 或使用者 .bashrc，需要在 service 檔的 Environment= 裡設定
- Docker socket 掛載才能讓容器內的 hermes 操作 Docker，否則只能操作主機
